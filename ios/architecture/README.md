# iOS 아키텍처 패턴

모든 타겟 회사가 MVVM을 필수로 요구하고, 86%가 Clean Architecture를 요구합니다.

## 폴더 구조 & 학습 순서

### 필수 (Lv3~4)
| 순서 | 폴더 | 핵심 | 면접 빈출도 |
|------|------|------|------------|
| 1 | `mvc/` | Apple MVC, 한계점 이해 | ★★★ |
| 2 | `mvvm/` | MVVM + 데이터 바인딩 | ★★★★★ |
| 3 | `clean-architecture/` | 레이어 분리, DI, UseCase | ★★★★★ |
| 4 | `coordinator/` | 화면전환 로직 분리 | ★★★★ |

### 차별화 (Lv2: 이해+설명)
| 순서 | 폴더 | 핵심 | 타겟 회사 |
|------|------|------|----------|
| 5 | `advanced-patterns/` | TCA, RIBs, ReactorKit, VIPER | 하이퍼커넥트, 오늘의집, 카카오뱅크 |
| 6 | `modularization/` | 모듈화 전략, 마이크로 피처 | 현대차, 당근, 토스 |

## 정리 방식
각 패턴마다:
1. **개념 다이어그램** (텍스트로 그려도 OK)
2. **간단한 예제 프로젝트 코드**
3. **장단점 & 언제 사용하는지**
4. **SUSA24에서 어떻게 적용했는지** (면접 답변용)

- Clean Architecture의 레이어를 설명하세요 (Presentation → Domain → Data)

- 본인 프로젝트에서 아키텍처를 선택한 이유는? (SUSA24의 Redux 패턴)
